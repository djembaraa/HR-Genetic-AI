# Database Relationships - HR-Genetic-AI

## 1. Relational Database (SQLite via Prisma)
Used for structured, transactional data.

### Models
**Job**
- `id` (Int, PK, AutoIncrement)
- `title` (String)
- `department` (String)
- `description` (Text)
- `created_at` (DateTime)

**Candidate**
- `id` (Int, PK, AutoIncrement)
- `name` (String)
- `email` (String, Unique)
- `cv_url` (String) - Path to the stored file
- `applied_job_id` (Int, FK -> Job.id)
- `created_at` (DateTime)

### Relationships
- A `Job` can have many `Candidates` (1-to-N).
- A `Candidate` applies to one `Job`.

## 2. Vector Database (ChromaDB)
Used for semantic search and AI RAG.

### Collection: `cv_embeddings`
- **ID:** Auto-generated UUID for each text chunk.
- **Document:** The actual string text of the CV chunk.
- **Embedding:** Vector array (generated by Gemini).
- **Metadata:**
  - `candidate_id` (Maps directly to `Candidate.id` in SQLite)
  - `page_number` (For reference)

*By linking the ChromaDB metadata to the SQLite Primary Keys, the AI can cross-reference vector search results with actual relational data.*
